# Shatakshi Herbal

An Expo React Native app for a herbal products store (Ayush-certified, 100% natural). Includes a product catalogue, category browsing, and featured products. Backed by an Express 5 API server with PostgreSQL + Drizzle ORM.

## Run & Operate

- **Mobile app (primary):** managed by the `artifacts/mobile: expo` workflow — uses `pnpm --filter @workspace/mobile run dev`
- **API server:** managed by the `artifacts/api-server: API Server` workflow — uses `pnpm --filter @workspace/api-server run dev` (port 5000)
- **Mockup sandbox:** managed by the `artifacts/mockup-sandbox: Component Preview Server` workflow — Vite dev server for UI component previews
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)

## Required environment variables

- `DATABASE_URL` — Postgres connection string (required by API server)
- `PORT` — injected automatically by Replit per workflow
- `REPLIT_DEV_DOMAIN`, `REPLIT_EXPO_DEV_DOMAIN`, `REPL_ID` — injected automatically by Replit (used by mobile dev script)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Mobile: Expo ~54, React Native 0.81, expo-router ~6
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `artifacts/mobile/` — Expo React Native app (Shatakshi Herbal storefront)
- `artifacts/api-server/` — Express 5 REST API
- `artifacts/mockup-sandbox/` — Vite component preview server (design tooling)
- `lib/api-client-react/` — React Query hooks (generated by Orval)
- `lib/api-spec/` — OpenAPI spec (source of truth for API shape)
- `lib/api-zod/` — Zod schemas (generated by Orval from spec)
- `lib/db/` — Drizzle ORM schema and DB client

## Architecture decisions

- API shape is owned by the OpenAPI spec in `lib/api-spec/`; client hooks and Zod validators are code-generated — never edit generated files directly
- Mobile uses expo-router for file-system based routing
- React Compiler is enabled in the mobile app (babel plugin)
